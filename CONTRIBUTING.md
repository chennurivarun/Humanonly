# Contributing to HumanOnly

Thanks for helping build human-first social infrastructure.

## First Steps
1. Read `MANIFESTO.md` and `GOVERNANCE.md`
2. Pick an issue labeled `good-first-issue` or `help-wanted`
3. Open a draft PR early for feedback

## Branch Naming
- `feat/<scope>-<short-name>`
- `fix/<scope>-<short-name>`
- `docs/<scope>-<short-name>`

## PR Requirements
- Clear problem statement
- Decision rationale
- Tests/docs updated
- No hidden behavior changes

## Safety
Do not implement features that bypass moderation audit logs or human override controls.
